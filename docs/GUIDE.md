## Что такое XAM Messenger

XAM Messenger — это корпоративный мессенджер для работы в локальной сети.

**Особенности:**
- 🔒 **Только локальная сеть** — без интернета, полная приватность
- 🚀 **Быстрый старт** — авто-обнаружение сервера
- 📁 **Файлообмен** — отправка файлов до 100MB
- ✓✓ **Статусы** — доставлено / прочитано
- 💾 **История** — SQLite для хранения сообщений
- 🖥️ **Универсальный клиент** — браузер + Tauri приложение

## Установка

### Вариант 1: Готовые сборки

Скачайте сборку для вашей платформы из [Releases](https://github.com/dev-lime/xam/releases):

| Платформа | Файл |
|-----------|------|
| **Windows** | `xam-messenger-windows-x64.msi` (установщик) |
| **Windows Portable** | `xam-messenger-portable-windows-x64.exe` |
| **Linux (Debian/Ubuntu)** | `xam-messenger-linux-x64.deb` |
| **Linux (Fedora/openSUSE)** | `xam-messenger-linux-x64.rpm` |
| **Linux (Universal)** | `xam-messenger-linux-x64.AppImage` |
| **macOS (Intel + Apple Silicon)** | `xam-messenger-macos-universal.dmg` |

### Вариант 2: Сборка из исходников

Требования:
- Rust 1.75+
- Node.js 18+
- Tauri 2.0+

```bash
# Клонируйте репозиторий
git clone https://github.com/dev-lime/xam.git
cd xam

# Соберите сервер
cd server
cargo build --release

# Соберите клиент (Tauri)
cd ../src-tauri
cargo tauri build
```

## Быстрый старт

### 1. Запуск сервера

**Linux/macOS:**
```bash
cd server
./target/release/xam-server
```

**Windows:**
```powershell
cd server
.\target\release\xam-server.exe
```

**Сервер запустится на порту 8080.**

### 2. Запуск клиента

**В браузере:**
```bash
open src/index.html
```

**Tauri приложение:**
```bash
cd src-tauri
cargo tauri dev
```

### 3. Подключение

1. Введите ваше имя
2. Нажмите **"Войти"**
3. Сервер найдётся автоматически ✨

## Использование

### Регистрация

При первом входе введите имя (логин). Имя должно быть уникальным в пределах сервера.

### Отправка сообщений

1. Выберите контакт из списка слева
2. Введите сообщение в поле ввода
3. Нажмите Enter или кнопку отправки

### Выбор сервера

Если сервер не найден автоматически:

1. Нажмите кнопку **"🔄 Сменить сервер"** в меню профиля
2. Введите IP адрес сервера вручную (например, `192.168.1.100:8080`)
3. Нажмите **"Подключиться"**

### Профиль

Для изменения профиля:

1. Нажмите на аватар в левом верхнем углу
2. Выберите **"Профиль"**
3. Измените имя или аватар (эмодзи)
4. Нажмите **"Сохранить"**

## Отправка файлов

### Как отправить файл

1. Нажмите кнопку **📎** слева от поля ввода
2. Выберите файл(ы)
3. Файлы прикрепятся к сообщению
4. Отправьте сообщение

### Лимиты

- **Максимальный размер:** 100MB
- **Типы файлов:** любые
- **Количество:** несколько файлов в одном сообщении

### Получение файлов

1. Кликните на имя файла в сообщении
2. Файл скачается в папку загрузок

## Решение проблем

### Сервер не найден

**Проблема:** Клиент не может найти сервер в локальной сети.

**Решение:**

1. **Убедитесь что сервер запущен:**
   ```bash
   ./server/target/release/xam-server
   ```

2. **Проверьте firewall:**

   **Windows (от имени администратора):**
   ```powershell
   New-NetFirewallRule -DisplayName "XAM Messenger Server" -Direction Inbound -Protocol TCP -LocalPort 8080 -Action Allow
   ```

   **Linux:**
   ```bash
   sudo ufw allow 8080/tcp
   ```

   **macOS:** Системные настройки → Защита и безопасность → Фаервол → Добавьте `xam-server`

3. **Узнайте IP сервера:**

   **macOS:**
   ```bash
   ipconfig getifaddr en0
   ```

   **Linux:**
   ```bash
   hostname -I | awk '{print $1}'
   ```

   **Windows:**
   ```powershell
   ipconfig | findstr IPv4
   ```

4. **Попробуйте ручной ввод адреса:**
   - Нажмите "🔄 Сменить сервер"
   - Введите IP адрес сервера
   - Нажмите "Подключиться"

---

### Ошибка при загрузке файла

**Проблема:** Файл не загружается или превышает лимит.

**Решение:**
- Проверьте размер файла (макс. 100MB)
- Проверьте свободное место на сервере
- Попробуйте отправить файл меньшего размера

---

### Сообщения не доставляются

**Проблема:** Сообщения остаются в статусе "Отправка".

**Решение:**
- Проверьте подключение к сети
- Убедитесь что получатель онлайн
- Перезапустите клиент

---

### mDNS не работает

**Проблема:** Сервер не обнаруживается автоматически.

**Причины:**
- mDNS заблокирован firewall
- Сетевые политики запрещают multicast
- Разные подсети у клиента и сервера

**Решение:** Используйте ручной ввод IP адреса.

## Конфигурация сервера

Сервер поддерживает переменные окружения для настройки:

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `XAM_HOST` | `0.0.0.0` | Хост для прослушивания |
| `XAM_PORT` | `8080` | Порт для прослушивания |
| `XAM_DB_PATH` | `~/.config/xam-messenger/xam.db` | Путь к базе данных |
| `XAM_UPLOAD_DIR` | `~/.local/share/xam-messenger/files` | Директория загрузок |
| `MAX_FILE_SIZE` | `104857600` | Макс. размер файла (100MB) |
| `RUST_LOG` | `info` | Уровень логирования |

**Пример запуска с настройками:**
```bash
XAM_PORT=9000 RUST_LOG=debug ./target/release/xam-server
```
